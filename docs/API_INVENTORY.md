# Strudel → Rudel API inventory

Generated by `crates/rudel-lang/tests/api_inventory.rs` — do not edit by hand. The Strudel side is `tools/oracle/reference_golden.json` (jsdoc `@name`/`@synonyms` + `register`/`registerControl`, per package); the Rudel side is `rudel_lang::reference()` (live runtime introspection). Each documented Strudel name is classified against `tools/oracle/reference_allowlist.json`:

- **implemented** — exposed by Rudel.
- **intentional** — intentionally different or not applicable (allowlist status `unsupported`).
- **deferred** — a tracked gap owned by another checklist item (allowlist status `deferred`).
- **unaccounted** — neither implemented nor documented; this must never appear.

Regenerate with `RUDEL_BLESS=1 cargo test -p rudel-lang --test api_inventory`. A name defined in more than one Strudel package is listed under each.
